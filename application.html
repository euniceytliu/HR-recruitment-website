🤖 案例一：搭建AI驱动的智能简历筛选工具
中等难度
背景：腾讯某事业群校招面临挑战：简历量大（5000+）但HR人手有限，优质候选人识别效率低，JD与简历匹配准确率不足，导致面试转化率仅18%。团队希望引入AI提升招聘效率和精准度。

🎯 必须交付的实用产品：
构建一个可运行的"AI简历筛选工具"（最小化产品）
  • 核心功能：上传简历（PDF/Word）→ AI自动评分（0-100分）→ 输出匹配度报告（优势/不足/面试建议）
  • 技术要求：可使用任何AI工具（ChatGPT、Claude、文心一言等），需提供完整提示词
  • 可交付形式：①网页工具（HTML+JS）或 ②Google Colab/Jupyter笔记本 或 ③可运行的Python脚本
  • 测试要求：至少用3份真实简历测试，展示筛选结果

📋 产品开发任务（完整流程）：
  1. 问题定义：明确你的工具要解决什么问题？输入是什么（简历格式、岗位JD）？输出是什么（评分标准、报告格式）？
  2. 任务拆解：将产品开发拆解为4-6个可执行步骤（如：简历解析 → 关键信息提取 → AI评分 → 报告生成），每步明确输入/输出
  3. AI提示词设计：撰写核心提示词（至少包含：简历解析、匹配度评分、面试建议生成），并展示你如何通过多轮测试优化提示词
  4. 产品搭建：动手构建最小化产品，确保功能可用（上传简历 → 得到评分报告）
  5. 测试与优化：用至少3份真实简历测试，记录问题并优化（如：AI评分偏高/偏低、报告不够具体等）
  6. 方案验证：设计评估标准（准确性、实用性、效率提升），说明如何验证工具是否真正解决了问题
⏰ 建议用时：6-8小时 🎯 考察能力：产品思维、AI协同、动手能力、全流程交付

💡 案例二：开发AI个性化能力诊断与发展助手
高难度
背景：腾讯HR发现员工成长路径不清晰，能力评估主观性强，培训资源匹配不精准。管理者反馈：不知道如何针对性培养团队成员。公司希望构建AI驱动的个性化能力发展体系。

🎯 必须交付的实用产品：
构建一个可运行的"AI能力诊断与发展助手"（最小化产品）
  • 核心功能：用户输入岗位和当前能力情况 → AI诊断能力短板 → 推荐个性化成长路径（3-6个月计划）→ 匹配学习资源
  • 技术要求：可使用任何AI工具，需设计多轮对话流程（至少3轮），让AI逐步深入了解用户情况
  • 可交付形式：①对话式网页工具 或 ②ChatGPT/Claude自定义GPT 或 ③可运行的聊天机器人脚本
  • 测试要求：至少用3个不同岗位场景测试（如：产品经理、数据分析师、技术支持），展示完整对话和推荐结果

📋 产品开发任务（完整流程）：
  1. 问题定义：将"员工发展不清晰"拆解为AI可解决的子问题（能力诊断、路径推荐、资源匹配），明确每个子问题的输入/输出和评判标准
  2. 逻辑框架设计：设计AI能力评估的逻辑（评估维度、诊断标准、推荐依据），画出系统流程图（用户输入 → AI处理 → 结果输出）
  3. 多轮对话设计：设计AI与用户的对话流程（至少3轮），每轮对话的目标是什么？如何引导用户提供关键信息？如何处理用户回答不清晰的情况？
  4. 提示词工程：撰写核心提示词（能力诊断、路径推荐、资源匹配），展示你如何通过测试迭代优化提示词（至少展示2次迭代过程）
  5. 产品搭建：动手构建最小化产品，确保对话流程顺畅、推荐结果个性化
  6. 测试与验证：用3个不同岗位测试，记录AI推荐的成长路径是否合理、学习资源是否匹配、是否真正有实用价值
  7. 方案优化：识别产品不足（如：推荐太泛化、资源不够具体等），说明如何改进
⏰ 建议用时：8-10小时 🎯 考察能力：系统设计、对话设计、AI协同、产品思维、全流程交付

📊 案例三：构建AI驱动的HR数据洞察仪表盘
入门难度
背景：腾讯HR每月需要输出大量数据报告（离职率、招聘效率、员工满意度等），但数据分散在多个系统，手动整理耗时且容易出错。管理层希望HR能从数据中洞察业务问题并给出决策建议，而非仅提供数字。

🎯 必须交付的实用产品：
构建一个可运行的"AI数据洞察工具"（最小化产品）
  • 核心功能：上传HR数据（Excel/CSV）→ AI自动分析（识别异常、发现趋势）→ 生成洞察报告（关键发现+业务建议）
  • 技术要求：可使用任何AI工具+数据处理工具（Python、Excel、Google Sheets等），需提供完整提示词和数据处理流程
  • 可交付形式：①网页工具（可视化仪表盘）或 ②Google Colab/Jupyter笔记本 或 ③可运行的分析脚本+报告模板
  • 测试要求：至少用2组真实/模拟HR数据测试（如：离职数据、招聘数据），展示AI生成的洞察报告

📋 产品开发任务（完整流程）：
  1. 问题定义：选择1个具体的数据分析场景（如：离职分析、招聘效率分析），明确输入数据格式、分析目标、输出报告内容
  2. 任务拆解：将数据分析拆解为3-5个步骤（如：数据清洗 → 描述性统计 → 异常识别 → AI洞察生成 → 报告输出）
  3. AI提示词设计：撰写2个核心提示词：①让AI从数据中发现异常模式；②让AI根据数据趋势给出业务建议。展示你如何优化提示词让AI输出更有价值
  4. 产品搭建：动手构建最小化产品，确保数据能顺利导入、AI能自动分析、报告能清晰展示
  5. 测试与验证：用2组数据测试，检查AI洞察是否准确（数据解读是否正确）、是否有用（建议是否可执行）
  6. 质量评估：设计5个检查点评估AI报告质量（如：数据准确性、逻辑合理性、结论可操作性），说明如何判断报告是否可用
  7. 快速学习：如果你此前没用过AI做数据分析，说明你如何在3天内完成这个产品（学习路径+时间分配+关键资源）
⏰ 建议用时：4-6小时 🎯 考察能力：数据思维、AI协同、动手能力、快速学习、全流程交付
