# 📊 案例评估AI逻辑说明

## 🎯 总览

案例评估系统使用**智谱AI (GLM-4)** 根据预定义的评分标准对学生提交的案例进行自动评估。

---

## 📋 核心配置文件

### **1. `hr-evaluation-config.js` - 评估标准配置**
定义了完整的评估逻辑和参考标准。

### **2. `case-evaluation-ai.html` - AI评估执行页面**
HR管理页面，包含完整的AI提示词和评估逻辑。

### **3. `auto-evaluation-config.js` - 自动评估开关**
控制是否在学生提交后自动触发评估（当前关闭）。

---

## 🎛️ 评分标准详解（总分105分）

### **一、提交材料完整性（20分）**

| 维度 | 分数 | 说明 |
|------|------|------|
| 可运行的产品 | 5分 | 产品链接有效且可访问 |
| 演示视频 | 3分 | 提供演示视频链接 |
| 核心AI提示词 | 4分 | 提示词内容完整专业 |
| 测试数据与结果 | 4分 | 提供测试案例和结果 |
| 说明文档 | 4分 | 上传详细说明文档 |

**扣分规则：**
- ❌ 产品链接无效或为测试链接 → 扣5分
- ❌ 未提供视频 → 扣3分
- ❌ 提示词少于100字 → 扣2-4分
- ❌ 未提供测试数据 → 扣4分
- ❌ 未上传文档 → 扣4分

---

### **二、产品功能与质量（30分）**

| 维度 | 分数 | 评估标准 |
|------|------|---------|
| 核心功能实现 | 15分 | 优秀：13-15分<br>良好：10-12分<br>及格：7-9分 |
| AI效果质量 | 15分 | 优秀：13-15分<br>良好：10-12分<br>及格：7-9分 |

**评估要点：**
- ✅ 产品是否真正解决了问题？
- ✅ 核心功能是否完整可用？
- ✅ AI的输出质量如何？
- ✅ 用户体验是否良好？

**扣分规则：**
- ❌ 产品说明少于200字 → 功能质量不超过10分（满分30）
- ❌ 产品功能不完整或有明显bug → 扣5-10分
- ❌ AI效果差或不稳定 → 扣5-10分

---

### **三、AI协同能力（25分）**

| 维度 | 分数 | 评估标准 |
|------|------|---------|
| 问题定义能力 | 8分 | 优秀：7-8分<br>良好：5-6分<br>及格：3-4分 |
| AI提示词设计 | 10分 | 明确性、结构性、迭代优化 |
| 方案判断能力 | 7分 | 优秀：6-7分<br>良好：4-5分<br>及格：2-3分 |

**评估要点：**
- ✅ 是否清晰定义了要解决的问题？
- ✅ 提示词是否结构化、专业化？
- ✅ 是否经过多轮测试和优化？
- ✅ 是否能合理判断AI输出的质量？

**扣分规则：**
- ❌ 提示词少于50字或为测试内容 → 此维度不超过8分（满分25）
- ❌ 提示词结构混乱、不专业 → 扣3-5分
- ❌ 未体现迭代优化过程 → 扣2-4分

---

### **四、全流程交付能力（15分）**

| 维度 | 分数 | 说明 |
|------|------|------|
| 任务拆解 | 5分 | 是否将复杂任务拆解为可执行步骤 |
| 测试验证 | 5分 | 是否进行充分的测试和验证 |
| 问题识别与优化 | 5分 | 是否发现问题并优化改进 |

**评估要点：**
- ✅ 是否有清晰的任务拆解（4-6步）？
- ✅ 是否用真实数据测试？
- ✅ 是否记录问题并优化？
- ✅ 是否验证最终方案的有效性？

---

### **五、学习能力与创新性（10分）**

| 维度 | 分数 | 说明 |
|------|------|------|
| 快速学习能力 | 5分 | 短时间内掌握新工具新方法 |
| 创新性 | 5分 | 独特的思路或巧妙的解决方案 |

**评估要点：**
- ✅ 是否在短时间内学会使用AI工具？
- ✅ 是否有创新的应用场景或方法？
- ✅ 是否有独特的见解或优化思路？

---

### **六、文档质量（加分项，最高+5分）**

| 维度 | 分数 | 说明 |
|------|------|------|
| 文档结构清晰 | +2分 | 目录清晰、逻辑顺畅 |
| 丰富的可视化 | +2分 | 流程图、截图、数据图表 |
| 逻辑严密专业 | +1分 | 语言专业、论证充分 |

## 🤖 AI评估提示词（Prompt）

### **核心结构**

```
你是腾讯HR的专业案例评估专家。请严格按照以下评估标准对学生提交的案例进行评分。

⚠️ 重要提醒：
- 你无法真正访问链接，只能根据文字描述评估
- 如果提示词内容过于简单（如"test"、"测试"等），必须大幅扣分
- 如果产品说明不详细，无法判断功能质量，必须给予低分
- 评分必须严格，宁可偏严不可偏松

# 评估标准（总分100分）
[详细的6大维度评分标准]

# 学生提交的案例信息
- 案例名称
- 产品链接（注意：AI无法访问）
- 演示视频（注意：AI无法观看）
- 核心AI提示词（字数统计）
- 产品说明与测试结果（字数统计）
- 是否提交说明文档

# 严格评分要求
1. 如果提示词少于50字或内容为测试内容 → "AI协同能力"不超过8分（满分25）
2. 如果产品说明少于100字 → "产品功能与质量"不超过10分（满分30）
3. 如果链接明显为测试链接（含test等）→ "提交材料完整性"不超过10分（满分20）
4. 总分必须合理：材料严重不足的情况下，总分不应超过50分

# 输出格式
{
  "totalScore": 85,
  "dimensions": {
    "completeness": { "score": 18, "maxScore": 20, "comment": "..." },
    "product": { "score": 25, "maxScore": 30, "comment": "..." },
    "aiCollaboration": { "score": 20, "maxScore": 25, "comment": "..." },
    "delivery": { "score": 12, "maxScore": 15, "comment": "..." },
    "learning": { "score": 8, "maxScore": 10, "comment": "..." },
    "documentation": { "score": 2, "maxScore": 5, "comment": "..." }
  },
  "strengths": ["优势1", "优势2", "优势3"],
  "improvements": ["不足1", "不足2", "不足3"],
  "suggestions": ["建议1", "建议2", "建议3"],
  "summary": "一句话总结评价"
}
```

## ⚙️ AI模型配置

### **使用模型：智谱AI GLM-4**

```javascript
model: 'glm-4-flash',
temperature: 0.3,  // 低温度确保评分稳定
max_tokens: 3000   // 支持详细反馈
```

### **为什么选择GLM-4？**
- ✅ 中文理解能力强
- ✅ 结构化输出稳定
- ✅ 成本合理（每次评估约0.01元）
- ✅ 响应速度快（3-5秒）

## 🛡️ 防作弊机制

### **1. 前置检查（提交时）**

```javascript
preChecks: {
  minUrlLength: 20,           // 链接最小长度
  minPromptsLength: 100,      // 提示词最小字数
  minSummaryLength: 200,      // 产品说明最小字数
  blacklistKeywords: ['test', '测试', 'demo', '示例']
}
```

### **2. AI智能识别（评估时）**

AI会自动检测：
- ❌ 测试链接（包含 test/demo 等）
- ❌ 过于简单的提示词（少于50字）
- ❌ 敷衍的产品说明（少于100字）
- ❌ 明显的复制粘贴内容

### **3. 严格扣分规则**

- 材料不完整 → 总分上限50分
- 提示词太简单 → AI协同能力上限8分（满分25）
- 产品说明太少 → 产品功能质量上限10分（满分30）

## 🎯 评估流程

### **手动评估流程（当前模式）**

```
1. 学生提交案例
   ↓
2. 数据保存到Supabase（status: pending）
   ↓
3. HR登录管理后台
   ↓
4. 查看案例详情
   ↓
5. 点击"🤖 AI评估"按钮
   ↓
6. AI分析案例（3-5秒）
   ↓
7. 显示AI评估结果（6大维度+总分）
   ↓
8. HR审核并可修改评分
   ↓
9. 点击"💾 保存评估"
   ↓
10. 更新数据库（status: reviewed）
   ↓
11. 学生可查询结果
```

### **自动评估流程（可启用）**

如果启用 `auto-evaluation-config.js` 中的 `enabled: true`：

```
1. 学生提交案例
   ↓
2. 前置检查（链接、字数、黑名单）
   ↓
3. 通过 → 延迟5秒后自动调用AI评估
   ↓
4. 失败 → 标记为 pending，等待手动评估
```

## 🔧 如何调整评估标准

### **方法1：修改配置文件（推荐）**

编辑 `hr-evaluation-config.js`：

```javascript
// 调整评分严格度
scoring: {
    strictness: 'strict',  // 改为严格模式
    excellent: 95,         // 优秀线提高到95
    good: 85,
    acceptable: 75,
    needImprove: 65
}

// 启用/禁用文化契合度评估
tencentValues: {
    enabled: true,
    weight: 15  // 提高权重到15%
}

// 添加自定义关键词
bonusKeywords: {
    tier1: ['腾讯', '阿里', '字节', '美团'],  // 添加美团
    tier2: ['HR实习', 'HRBP', '招聘实习'],
    tier3: ['Excel', 'PPT', '简历筛选']
}
```

### **方法2：直接修改Prompt**

编辑 `case-evaluation-ai.html` 第654行的 `prompt` 变量：

```javascript
const prompt = `你是腾讯HR的专业案例评估专家...
[修改评分标准]
[调整扣分规则]
[增加评估维度]
...`
```
